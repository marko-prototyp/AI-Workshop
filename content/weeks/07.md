---
num: "07"
title: "Build your design skill. Hand it off. Build the core view."
phase: "Make"
principle: "Your direction becomes a skill. Give the model a reference image and one sharp frame, and every screen after it inherits your look instead of the model's default."
capability: "Turn a reference image into an installable design skill; cold-test it; write a build brief for Claude Code; build the core view"
aim: "Turn one reference image into your design skill, prove it builds your look, then hand it to Claude Code to build the first screen."
toolkit: "Claude chat · Claude Code"
tools:
  - Claude chat
  - Claude Code
flow:
  - Make the skill
  - Test it cold
  - Write the deliverable
  - Build it
before: "Everything you've made so far lives in your project: the design system, the wireframes, the real content, and your anti-stock skill. Have your W04 wireframe for the first screen ready, and grab the three skills below. You'll fill the template into your own design skill this hour."
installs:
  - name: design-direction-template
    blurb: "The scaffold you fill into your own design skill, from one reference image."
    file: design-direction-template.md
  - name: style-extract
    blurb: "Reads a reference for its style, banned moves, and AI-tells."
    file: style-extract.md
  - name: system-extract
    blurb: "Reads a reference for its tokens and type scale."
    file: system-extract.md
exercise:
  - "Fill the design-direction template into your own design skill from one reference image."
  - "Test the skill cold: drop your wireframe, generate one screen, no rules attached."
  - "Write a short build brief for Claude Code."
  - "Build the core view in Claude Code, one section at a time."
prompts:
  - title: "Fill the template into your design skill"
    tool: "Claude chat"
    does: "Hand Claude one reference image and the design-direction template. It fills every slot with your real values and packages an installable skill."
    preamble: "What's the one owned move in your reference you can point at and name? Pick one image, not a board of ten."
    body: |
      I want to turn one reference image into my project's design skill, using the design-direction template.

      My reference image: [attach the one image that nails the look]
      The template: [paste the design-direction template, or attach the file]
      My project: [name] ([one line on what it is])

      Fill the template into a real skill, in this order, and pause after each part so I can check it:
      1. Sharpen the one-line frame first: "It is a [X], not a [Y]." A vague frame can't be fixed later, so push back until it's sharp.
      2. Read the reference and fill every slot: the :root tokens, the type scale, the banned moves, the grounding. Pin exact values from the image. Where the image can't tell you a value, mark it TODO. Don't invent it.
      3. Rename it [project]-design, write the description using my real screen names as triggers, and delete the fill-in instructions.
      4. Package it as a .skill file and tell me how to install it.

      Copy the tokens and the frame verbatim once they're set. Don't re-derive them later.
    note: "The template recommends two helpers: run style-extract on the reference for the style and banned moves, then system-extract for the tokens and the type scale, and fill the template from both. Skip them and Claude reads the image directly. Either way, the reference image wins over any description."
    gives: "[project]-design: an installed skill holding your frame, your tokens, and your banned moves."
  - title: "Test the skill cold"
    tool: "Claude chat"
    does: "Open a fresh chat, drop in your wireframe, and ask for one screen with no rules attached. You haven't coached it, so this is the real test of the skill."
    preamble: "Watch four things, all on their own: does the skill trigger, hold your tokens, keep your frame, and clear your banned moves, without you saying a word?"
    body: |
      Build the [first screen, for example the roster row] for [project]. [attach or point to your W04 wireframe for this screen]
    note: "If it holds, that's your first screen. If it drifted, fix the skill, not the chat: \"The build drifted here: [name what slipped]. Update the SKILL.md to close that gap, then tell me how to re-install.\" A drift means a missing banned move or a vague frame."
    gives: "Your first screen, and a straight answer on whether the skill holds without you in the room."
  - title: "Write the deliverable for Claude Code"
    tool: "Claude chat"
    does: "Pack one short build brief Code can work from: what to read first, what's non-negotiable, and the build order. It points at your skill, it doesn't re-paste it."
    preamble: "If Code had only this one message, could it build your screen? What's missing that isn't already in your project?"
    body: |
      I'm about to hand my first screen to Claude Code. Write me one short build brief it can work from.

      Everything durable is already in my project: the design system, my [project]-design skill, the wireframes, and the real content. The screen I'm building first is [name it]. The version I approved in the cold test is here: [attach the screen].

      Do this:
      1. Write a short README that tells Code what to read first and what's non-negotiable: my [project]-design skill and the banned moves.
      2. Break this screen into a build order: which section first, second, third, and why.
      3. Flag anything thin or contradictory. Ask me one question at a time before writing that part.

      Output it as a single block I can save and point Code at. Reference my skill and design system by name. Don't paste them in, they're in the project.
    gives: "One build brief Code can build the whole screen from."
  - title: "Build the core view in Claude Code"
    tool: "Claude Code"
    does: "Hand Code the build brief and build the first screen, one section at a time, against your skill."
    preamble: "One section per prompt, never the whole page at once. Where does this screen connect to the rest of your app?"
    body: |
      Build [first screen] for [project].

      My build brief: [paste it, or point Code at the file].
      Use my [project]-design skill for the direction and the tokens, and match my wireframe and the screen I approved.

      Build the layout first, one section at a time, not the whole page at once. If something looks unusual, ask before smoothing it out. Don't add sections I didn't design.
    note: "When it's close but off, name the gap precisely: \"The [element] is off from the screen I approved. Specifically: [name it]. Fix only that. Don't rebuild the screen.\" Vague asks get vague changes."
    gives: "Your core view, built in Claude Code from your skill."
outputs:
  - "[project]-design: an installed design skill, filled from your reference."
  - "Your first screen, cold-tested against the skill."
  - "A build brief for Claude Code."
  - "The core view, built."
watch:
  - "One image, not a board of ten. Ten references average into nothing. Pick the one with a move you can name."
  - "Fixes go into the skill, not the chat. A drift means a missing banned move or a vague frame. Patch the SKILL.md and the next build inherits the fix."
  - "Your skill and design system live in your project. Point Code at them by name. You never paste them into a prompt."
  - "End the session with your check-in prompt. It interviews you in five questions and writes your report for you."
