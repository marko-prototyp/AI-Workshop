title: "Designing in Figma. Building with Claude."
phase: "Design → Build (Bridge)"
capability: "Figma-to-code workflow; image-as-reference; extracting design tokens from the build"
aim: "Your concept artifact validated the structure. Now make the visual real. What does the comp tell Claude that the artifact didn't?"
toolkit: "Sonnet in Claude.ai chat with Artifacts. Figma → build is pattern-matching at pace; save to disk for W07 if you want to keep iterating in Cowork."
exercise:
  - "Comp the main view in Figma at the device your real user will hold"
  - "Run the Figma → Claude prompt with the W05 artifact as structural reference"
  - "Iterate to match the comp's character"
  - "Extract tokens.css from what got built"
prompts:
  - title: "Figma → Claude (use this every time)"
    preamble: "What's the new information in the comp that the artifact didn't have? Style, exact type, real spacing. Your banned-moves list goes in here too."
    body: |
      [Attach Figma screenshot]

      I have a W05 concept artifact [link or paste] that validated the structure of this screen. Now build this Figma as a single HTML file at high fidelity. The artifact tells you what's on the screen and how it connects. The Figma tells you what it looks like.

      Important rules:

      1. Use these design tokens as CSS variables: [paste, or "extract from this build and define them"]
      2. Use these exact fonts: [Google Fonts links]
      3. Match spacing and proportions of the comp. Don't normalize.
      4. The visual direction is "[name]" — [3-word feel]
      5. Banned moves: [paste relevant items]
      6. Target device: [mobile / tablet / desktop — name what your real user holds]

      If something seems unusual, don't smooth it out. Ask me. Unusual is often the point. Don't add sections I didn't design.
  - title: "Refining a built section"
    preamble: "When Claude's close but not quite, what's the difference between asking for 'better' (vague) and naming the specific gaps (sharp)?"
    body: |
      The [section name] is close but not matching my Figma. Specifically:
      - [Difference 1]
      - [Difference 2]
      - [Difference 3]

      Make only these changes. Don't restructure. Don't normalise. If you think one of my asks is wrong, say so before changing it.
  - title: "Extract tokens from the build"
    preamble: "Claude built your screen but didn't formalise the system. Pull the tokens out so you can change one variable and see it propagate everywhere."
    body: |
      Look at the main view you just built from my Figma. Pull out the implicit design tokens — colors, type scale, spacing scale, radii — and give them back to me as a tokens.css using CSS custom properties. Use semantic names where it makes sense (e.g., --color-ink, --color-rule, --space-3). Comment the file so a designer reading it understands the system. If anything in the build feels off-system or inconsistent, flag it so I can decide whether to keep it or normalise it.
outputs:
  - "Figma comp of the main view at the device of your real user"
  - "High-fi build that captures the comp's character"
  - "tokens.css derived from the build"
watch:
  - "Design at the device your real user will hold. If they'll use this on a phone, comp at a phone. If on desktop, comp at desktop. Not always your laptop."
  - "If you need custom assets — illustrations, photos, sample content, sprites — generate or collect them this week, not at the end."
  - "Tokens out, not in. Don't formalise the system before you build."
  - "80% match is realistic. The designer fixes the last 20%."
