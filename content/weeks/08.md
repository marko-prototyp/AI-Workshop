prompts:
  - title: "Set up a repo, designer-style"
    preamble: "What's the part of git that feels foreign to you? Write a version of the prompt that asks for help with exactly that piece, before you copy the lifeline."
    body: |
      I'm a designer, not an engineer. I have a folder of HTML/CSS/JS files I built with Claude over the last few weeks. Walk me through, step by step, setting up a private GitHub repo for it. I'm using [GitHub Desktop / Claude Code / Terminal]. Don't assume command-line comfort. After each step, stop and confirm I did it before moving on. If I get a confusing error, ask me to paste it before suggesting a fix. The goal is one private repo with my project pushed, no more.
  - title: "Branch for an experiment"
    preamble: "What's something risky you've been wanting to try in your project? The branch is the permission to try it without losing the working version."
    body: |
      I want to try something risky with my project: [thing]. Before I do, set up a branch I can throw away if it doesn't work. Walk me through it in [GitHub Desktop / Claude Code]. Then explain how to swap back to the main branch if I want to give up. Treat me like someone who's never used git. After I'm on the branch, tell me how to know I'm on the branch (visual confirmation), so I don't accidentally commit experimental work to main.
  - title: "Read what changed"
    preamble: "When Claude breaks something subtle, where do you currently look to figure out what changed? The diff is the answer — but only if you can read one."
    body: |
      I made some changes to my project and something broke. I want to look at what changed since my last commit. Walk me through reading a diff in [GitHub Desktop / Claude Code]: what the red and green mean, what to focus on, what to ignore. Once I find the broken thing, tell me how to revert just that change without losing everything else I did since the last commit.

  - "One experimental branch made, kept or discarded"
