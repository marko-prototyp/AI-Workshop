title: "Designing in Figma, building with Claude"
phase: "Design → Build (Bridge)"
capability: "Figma-to-code workflow; image-as-reference; designer-led building"
demo: "The most important demo of the program. Take a Figma comp of a hero section, export it, paste into Claude with the standard template, watch Claude build, then refine specific parts. The lesson: design first."
exercise:
  - "Comp the main view in Figma — opinionated, not safe"
  - "Export at 2×"
  - "Hand to Claude with the template prompt"
  - "Iterate to match the comp's character"
prompts:
  - title: "Figma → Claude (use this every time)"
    body: |
      [Attach Figma screenshot]

      Build this as a single HTML file. Match the design closely. Important rules:

      1. Use these design tokens as CSS variables: [paste]
      2. Use these exact fonts: [Google Fonts links]
      3. Match spacing and proportions of the comp — don't normalize
      4. The visual direction is "[name]" — [3-word feel]
      5. Banned moves: [paste relevant items]

      If something seems unusual, don't smooth it out — ask me. Unusual is often the point. Don't add sections I didn't design.
  - title: "Refining a built section"
    body: |
      The [section name] is close but not matching my Figma. Specifically:
      - [Difference 1]
      - [Difference 2]
      - [Difference 3]

      Make only these changes. Don't restructure.
outputs:
  - "Figma comps for the main view"
  - "A built HTML version that captures the comp's character"
  - "Notes on where Claude got it right/wrong"
watch:
  - "If the Figma is bland, the build will be bland — push back on safe comps"
  - "Set expectations: 80% match is realistic; designer fixes the last 20%"
  - "Token use rises this week; worth it"
