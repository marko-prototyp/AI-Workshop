flow:
  - Make a repo
  - Branch an experiment
  - Read a diff
exercise:
  - "Set up a private repo and push your project."
  - "Make a branch for one experiment, then keep or discard it."
  - "Learn to read a diff and revert one thing."
prompts:
  - title: "Set up a repo, designer-style"
    tool: "GitHub Desktop / Claude Code"
    does: "Get your folder of files into a private repo, one careful step at a time."
    preamble: "What's the part of git that feels foreign to you? Hold that in mind as you go."
    body: |
      I'm a designer, not an engineer. I have a folder of HTML, CSS, and JS files I built with Claude over the last few weeks. Walk me through setting up a private GitHub repo for it, step by step, in [GitHub Desktop / Claude Code / Terminal]. Don't assume command-line comfort. After each step, stop and confirm I did it before moving on. If I hit a confusing error, ask me to paste it before suggesting a fix. The goal is one private repo with my project pushed, nothing more.
    note: "If your project pulls live data, make sure no API key gets committed. Ask Claude to check before your first push."
    gives: "One private repo with your whole project pushed."
  - title: "Branch for an experiment"
    tool: "GitHub Desktop / Claude Code"
    does: "A branch lets you try something risky without losing the working version."
    preamble: "What's the risky thing you've wanted to try but didn't dare? This is the safety net for it."
    body: |
      I want to try something risky with my project: [thing]. Before I do, set up a branch I can throw away if it doesn't work. Walk me through it in [GitHub Desktop / Claude Code]. Then show me how to switch back to main if I give up. Treat me like someone who's never used git. Once I'm on the branch, tell me how to confirm I'm on it, so I don't commit experimental work to main by accident.
    gives: "A safe branch to experiment on, and the way back."
  - title: "Read what changed"
    tool: "GitHub Desktop / Claude Code"
    does: "When something breaks, the diff tells you what changed. Learn to read one."
    preamble: "Right now, where do you look when Claude breaks something subtle? The diff is the real answer."
    body: |
      I made some changes and something broke. I want to see what changed since my last commit. Walk me through reading a diff in [GitHub Desktop / Claude Code]: what red and green mean, what to focus on, what to ignore. Once I find the broken thing, show me how to revert just that change without losing everything else since the last commit.
    gives: "The ability to see a diff and revert one thing, not everything."
outputs:
  - "A private repo, pushed."
  - "The commit habit started."
  - "One experimental branch, kept or discarded."
watch:
  - "This is preservation, not engineering. If it feels like a tutorial, that's fine."
  - "Catch anyone about to commit an API key."
